# 3 分钟 Demo 视频脚本

## 0:00-0:20 开场

展示首页 `AI 模拟面试官`。

口播：

> 这个产品面向准备技术实习面试的中国本科生。很多同学不缺题库，真正缺的是围绕自己简历和项目的连续追问，以及可执行复盘。

## 0:20-0:50 上传简历

点击 `PDF 样例简历`，加载一份 LaTeX 生成的脱敏 Agent/RAG 假简历。

口播：

> 现在主入口是上传简历，支持 PDF、DOCX 和 TXT。这里用的是一份 LaTeX 生成的假简历 PDF，不需要粘贴文字。文本 PDF 直接解析，如果是扫描版 PDF，会先尝试本地 OCR，识别不好时仍然可以粘贴文本兜底。

## 0:50-1:40 连续追问

选择 `RAG/Agent 项目真实性拷打`，点击 `开始面试`。

回答第一轮：

> 我用 500 字左右切块，重叠 80 字，先用 embedding 检索 top k，再用 rerank 重排，把片段交给大模型回答。

继续回答第二轮：

> 坏例主要是相邻章节召回混淆，我后来在 prompt 里要求引用课程章节。

口播：

> 它不是普通聊天，而是 Agentic RAG 工作流：每轮先检索面试资料卡，再结合简历证据生成追问。这里可以看到本轮依据，包括简历证据、风险假设、问题标签和资料来源。回答空泛时，它会继续要求具体数据、基线和失败路径。

## 1:40-2:10 场景切换

切到 `后端八股项目化追问`，点击 `开始面试`；再切回 `RAG/Agent 项目真实性拷打`。

口播：

> 同一份简历可以在多个场景下分别练习。每个训练场景都有独立对话，切回来还能继续原来的面试。

## 2:10-2:40 结束复盘

回到 RAG 场景，点击 `结束并复盘`，展示右侧结构化复盘。

口播：

> 复盘不是泛泛说“继续努力”，而是直接指出最可能被问挂的点，并按项目可信度、专业深度、表达结构、工程闭环和承压表现给出证据和改进方向。

## 2:40-3:00 收尾

展示公网 URL、模型状态和版本信息。

口播：

> 当前 demo 使用 DeepSeek V4-Pro，已部署在公网。浏览器只访问 3000 端口，后端通过同源 API 代理完成简历解析和面试追问。
