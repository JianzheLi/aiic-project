# 3 分钟 Demo 视频脚本

## 0:00-0:20 开场

展示首页 `AI 模拟面试官`。

口播：

> 这个产品面向准备技术实习面试的中国本科生。很多同学不缺题库，真正缺的是围绕自己项目经历的连续追问和可执行复盘。

## 0:20-0:50 输入项目

选择 `RAG/Agent 项目真实性拷打`，粘贴样例项目：

> 我做了一个基于 RAG 的课程问答系统，使用 FastAPI 提供后端接口，把课程 PDF 切分后写入向量库，通过 embedding 检索相关片段，再调用大模型生成回答。我主要负责后端接口、检索链路、Prompt 调优和 Docker 部署，做过一些坏例分析，比如相似课程章节被召回导致答案混淆。

点击 `开始面试`。

口播：

> 我们不要求用户先整理完整简历，只需要贴一段项目经历，降低开始训练的门槛。

## 0:50-1:40 连续追问

展示 AI 第一问，回答一段不够完整的内容，例如：

> 我主要按 500 字左右切块，重叠 80 字，先用 embedding 检索 top k，再把片段交给大模型回答。

继续回答第二轮：

> 坏例主要是相邻章节召回混淆，我后来在 prompt 里要求引用课程章节。

口播：

> 这里的关键是它不是普通聊天，而是像面试官一样抓住上一轮回答里的薄弱点继续追问，比如 chunk 怎么定、坏例怎么判断、问题出在检索还是生成。

## 1:40-2:25 结束复盘

点击 `结束并复盘`，展示右侧结构化复盘。

口播：

> 复盘不是泛泛说“继续努力”，而是直接指出最可能被问挂的点，并按项目可信度、专业深度、表达结构、工程闭环、承压表现给出证据和改进方向。

## 2:25-2:50 解释取舍

快速展示三个场景按钮和轮次状态。

口播：

> 16 小时内我们没有做登录、数据库、语音和复杂 agent 框架，而是优先做扎实的文字训练闭环：项目输入、连续追问、结构化复盘。

## 2:50-3:00 收尾

展示公网 URL 和版本信息。

口播：

> 当前 demo 已部署在公网，浏览器只访问 3000 端口，后端通过同源 API 代理调用模型。
