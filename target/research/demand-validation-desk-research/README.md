# 第三轮调研：需求验证 Desk Research

## 调研目标

考虑到挑战总时长只有 16 小时，当前不做真实用户访谈，改用公开资料快速验证三个问题：

1. 本科生或实习候选人常拿什么项目写进简历。
2. 当前后端、AI 应用、算法/LLM、AI Infra 实习 JD 真正在要求什么能力。
3. 通用聊天机器人、AI 面试产品、题库/面经平台为什么还不能完全解决这个场景。

## 样本统计

- 项目/简历样本：36 条。
- JD 样本：50 条。
- 竞品/替代方案体验：8 个。
- 原始结构化 JSON 数据保留在本地 `data/`，不提交到 Git；仓库只保留综合结论和产品判断。

## 样本边界

- 项目样本主要来自 GitHub 开源项目和公开项目页面，用于抽象“候选人可能写进简历的项目类型”，不记录个人简历隐私。
- JD 样本来自公开招聘页面、公司校招页面、招聘平台和高校就业附件，用于抽取技能信号，不保证岗位实时有效。
- 竞品体验使用公开产品页和可访问信息，只分析产品能力，不做商业排名。

## 主要产出

调研结论沉淀在 `findings.md`，最终被合并到 `target/research/synthesis.md` 和 `target/PLAN.md`。
