# 面试追问链 Playbook

## 使用原则

每条追问链都按“起手问题 -> 第一层追问 -> 第二层拷打 -> 判断标准 -> 常见挂点 -> 产品启发”组织。MVP 不需要完整题库，先把这些链做成可交互训练流程。

## 1. 项目真实性链

- 起手问题：你挑一个最能代表你能力的项目，2 分钟讲清楚。
- 第一层追问：背景是什么？你负责哪一块？为什么这块重要？
- 第二层拷打：这个方案是谁决定的？你有没有改过？如果去掉你，项目会少什么？
- 判断标准：能说清个人贡献、技术取舍、真实约束和可验证结果。
- 常见挂点：只背项目介绍；把团队成果说成个人成果；讲不清自己写的模块。
- 产品启发：用户输入项目后，AI 先做“真实性核验”，再进入八股追问。

## 2. Redis 项目链

- 起手问题：你项目里哪里用了 Redis？
- 第一层追问：为什么不用 MySQL 直接查？key/value 怎么设计？过期时间怎么定？
- 第二层拷打：缓存和数据库怎么一致？击穿/穿透/雪崩怎么处理？大 Key 和热点 Key 怎么排查？Redis 挂了怎么办？
- 判断标准：能把 Redis 从“会用”讲到数据结构、异常路径、监控和降级。
- 常见挂点：只会说“加缓存提高性能”；不会讲失败补偿。
- 产品启发：凡是项目里出现 Redis，默认生成缓存一致性和故障追问。

## 3. MySQL 索引链

- 起手问题：你项目里哪个查询最慢？怎么优化的？
- 第一层追问：建了什么索引？为什么这些列？Explain 看哪些字段？
- 第二层拷打：联合索引哪些列生效？什么情况下索引失效？B+树叶子/非叶子节点存什么？回表和索引下推怎么判断？
- 判断标准：能从 SQL、执行计划、数据量和业务读写模式解释优化。
- 常见挂点：只说“加索引”，不能解释为什么命中或失效。
- 产品启发：增加 SQL 诊断沙盘，让用户口述 explain 和优化依据。

## 4. MQ 一致性链

- 起手问题：你为什么引入 MQ？
- 第一层追问：消息什么时候发？发失败怎么办？消费失败怎么办？
- 第二层拷打：重复消费如何幂等？顺序性如何保证？消息堆积怎么排查？最终一致性怎么证明可接受？
- 判断标准：能覆盖正常路径、异常路径、补偿机制和业务容忍度。
- 常见挂点：只会说“异步解耦削峰”；不会处理失败和重试。
- 产品启发：训练回答必须包含“失败路径 + 补偿 + 监控”。

## 5. 性能压测链

- 起手问题：你说做过性能优化，指标是什么？
- 第一层追问：QPS、RT、错误率优化前后分别是多少？压测环境和线上差异是什么？
- 第二层拷打：瓶颈在线程池、DB、Redis、网络还是代码？单实例打满后怎么扩？多实例 TPS 反而下降怎么办？
- 判断标准：能给出指标、实验方法、瓶颈定位和回滚方案。
- 常见挂点：优化没有指标；只讲做了缓存或异步。
- 产品启发：项目卡片强制填写“优化前后指标”和“验证方法”。

## 6. 请求全链路链

- 起手问题：浏览器发起一个请求，到你的业务代码执行，中间经过什么？
- 第一层追问：DNS、TCP、HTTP、网关、Nginx、Servlet/SpringMVC 或 FastAPI 分别做什么？
- 第二层拷打：超时在哪一层配置？日志怎么串 trace？下游慢了怎么降级？
- 判断标准：能把网络、框架和业务串成一条可排障链路。
- 常见挂点：只会背 OSI/TCP，不会连接到自己的项目。
- 产品启发：做“请求链路白板题”，训练工程全局感。

## 7. 手撕算法链

- 起手问题：先说思路，再写代码。
- 第一层追问：复杂度是多少？边界条件有哪些？为什么这个数据结构合适？
- 第二层拷打：如果输入更大、含重复、内存受限、要返回路径，方案怎么变？
- 判断标准：能澄清题意、解释不变量、处理边界、从错误中定位。
- 常见挂点：模板题会写，变体和复杂度证明说不清。
- 产品启发：算法训练不是只判对错，要记录思路、边界和自测过程。

## 8. RAG 项目链

- 起手问题：你做的 RAG 项目数据从哪里来？
- 第一层追问：如何解析、切块、embedding、建索引？为什么这样切？chunk 多大？
- 第二层拷打：检索失败怎么定位？是数据、query rewrite、embedding、rerank 还是 prompt 的问题？有没有评估集？
- 判断标准：能讲清数据链路、召回/排序/生成边界、评估指标和坏例复盘。
- 常见挂点：只说“向量库 + 大模型”；没有评估和失败归因。
- 产品启发：AI 专项模拟必须包含坏例归因，而不是只问概念。

## 9. Agent 工程链

- 起手问题：你的 Agent 和普通 workflow 有什么区别？
- 第一层追问：任务怎么规划？工具怎么选择？状态和记忆存在哪里？
- 第二层拷打：工具超时怎么办？JSON 参数错怎么办？循环调用怎么检测？日志如何回放？
- 判断标准：能说清状态机、工具调用边界、异常处理和可观测性。
- 常见挂点：把 Agent 简化成“会调用工具的聊天机器人”。
- 产品启发：Agent 项目真实性可通过状态、恢复、失败重试来验证。

## 10. LLM 推理专项链

- 起手问题：KV Cache 为什么能加速推理？
- 第一层追问：prefill 和 decode 分别做什么？KV Cache 存在哪一层？显存怎么估算？
- 第二层拷打：vLLM/PagedAttention 解决什么问题？continuous batching 会带来什么 tradeoff？长上下文怎么影响吞吐？
- 判断标准：能从计算量、显存和调度解释推理系统。
- 常见挂点：只背框架名，不会算资源和解释瓶颈。
- 产品启发：AI Infra 面试要追到“资源、吞吐、延迟、显存”。

## 11. LoRA 微调链

- 起手问题：LoRA 为什么参数更少？
- 第一层追问：低秩矩阵插在哪里？rank 怎么选？冻结哪些参数？
- 第二层拷打：怎么合并部署？多 adapter 怎么管理？效果下降如何定位？
- 判断标准：能讲清低秩假设、参数效率、显存收益和部署影响。
- 常见挂点：只知道“省显存”，不知道训练和推理阶段差异。
- 产品启发：专项题要区分“用过库”和“理解原理/工程代价”。

## 12. 保研/导师压力链

- 起手问题：请用 2 分钟介绍你最重要的一段科研或项目经历。
- 第一层追问：你具体做了什么？问题意识是什么？为什么这个方法合适？
- 第二层拷打：如果结果不显著怎么办？你为什么适合我们课题组？如果没录取你会怎么规划？
- 判断标准：经历真实、表达稳定、方向匹配、不会的问题能诚实推理。
- 常见挂点：背稿、夸大经历、研究计划空泛。
- 产品启发：保研场景可以复用“经历真实性 + 匹配度 + 压力题反馈”。

## 13. 面试官识别背题链

- 起手问题：你刚才说熟悉这个技术，最近一次实际使用在哪里？
- 第一层追问：输入输出是什么？写了哪些代码？踩过什么坑？
- 第二层拷打：换一个约束还能做吗？如果线上失败，你第一步看哪里？
- 判断标准：能给出真实细节、代码边界、失败经验和迁移能力。
- 常见挂点：答案完美但模板化，细节追问后前后不一致。
- 产品启发：AI 面试官要避免只给“标准答案”，要制造真实追问压力。
